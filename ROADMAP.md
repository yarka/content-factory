# Roadmap

## Концепция развития

**Ядро** (platform-agnostic): идея → голос автора → проверенный черновик.
**Адаптеры** (platform-specific): каждая площадка получает свою версию из одного ядра.

```
                    ┌─────────────────┐
                    │   Core Draft    │
                    │  (твой голос)   │
                    └────────┬────────┘
                             │
          ┌──────────────────┼──────────────────┐
          ▼                  ▼                  ▼
    ┌───────────┐      ┌───────────┐      ┌───────────┐
    │ Telegram  │      │ LinkedIn  │      │  Threads  │
    │  адаптер  │      │  адаптер  │      │  адаптер  │
    └───────────┘      └───────────┘      └───────────┘
    RU, личное,        EN, бизнес,        RU/EN, коротко,
    до 1500 симв.      польза, факты      авто-репост
```

---

## Сейчас работает ✅

- **Telegram pipeline** — полный цикл от идеи до публикации
- **LinkedIn pipeline** — DNA настроен, адаптер готов (EN, 600-1200 симв, без хэштегов)
- **Онбординг** — основной канал + Platform Setup для каждой платформы
- **Research** — два режима: авто (Exa MCP) и референсы
- **Fact-check** — проверка фактов через web-поиск
- **Deaify** — 4 параллельных критика, правило коротких тире
- **Публикация** — staging / production через publish.py
- **Заголовок-хук** — 3 варианта (Zinger / First-Person / Question / Scene)

## Дальше — Competitor Intelligence (идеи когда не знаешь о чём писать)

**Идея:** когда нет идеи для поста — система сканирует что пишут релевантные авторы в нише и предлагает углы.

**Триггер:** `"о чём писать"`, `"дай идеи"`, `"что сейчас пишут"`

**Как работает:**
1. Берёт список авторов/аккаунтов из `competitors.md` (Telegram-каналы, LinkedIn-профили)
2. Ищет через Exa MCP их свежие посты и тренды в нише
3. Анализирует: что заходит, какие темы горячие, где незакрытый угол
4. Предлагает 3-5 идей для поста с твоим уникальным углом поверх

**Что нужно сделать:**
1. `competitors.md` — список авторов по платформам (Telegram, LinkedIn, X)
2. `rules/idea-generator.md` — правило: поиск → анализ → 5 идей с углом
3. Триггер в `CLAUDE.md`

## Дальше — Analytics Layer (учиться на своих постах)

**Идея:** после каждого поста — разбор. После 10+ постов — паттерны. Система учится что работает именно у тебя.

**Два уровня:**

**Уровень 1 — Пост-анализ (сразу после публикации)**
- Что за тип поста (история / инструмент / тренд / рефлексия)
- Какой хук использован
- Платформа, рубрика, длина
- Пишется автоматически в `output/analytics/posts-log.md`

**Уровень 2 — Периодический разбор (раз в месяц)**
- Триггер: `"разбор постов"`, `"что работает"`
- Анализирует `output/posts/` и `output/analytics/posts-log.md`
- Находит паттерны: какие хуки заходят, какие рубрики, какая длина
- Предлагает обновить `channel-dna.md` или `writing-guide.md` на основе данных

**Что нужно сделать:**
1. Обновить frontmatter в шаблоне поста (тип, хук, платформа, рубрика)
2. `output/analytics/posts-log.md` — журнал всех постов
3. `rules/analytics.md` — правило анализа паттернов
4. Триггер в `CLAUDE.md`

## Дальше — Картинки в едином стиле

**Идея:** к каждому посту генерировать обложку / карточку в едином визуальном стиле.

**Варианты реализации:**
- DALL-E / Flux через API — генерация по описанию поста
- HTML → скриншот (Playwright) — программный дизайн без нейросетей
- Canva API — шаблоны с переменными

**Что нужно сделать:**
1. Определить стиль (цвета, шрифт, сетка)
2. Выбрать инструмент генерации
3. Добавить фазу `visualize` в pipeline после Write

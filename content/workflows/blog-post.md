# Blog Post Pipeline

Модульный pipeline для написания статей: research → draft → deaify → publish.

## Quick Reference

| Что | Где |
|-----|-----|
| **Стиль** | `content/writing-guide.md` |
| **Термины AI** | `content/ai-terms-ru.md` |
| **Деаификация** | `content/workflows/deaify-text.md` |
| **Публикация** | `publish/workflows/publish.md` |
| **Конфиг** | `config/config.yaml` |

## Workflow

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  Phase 1:   │ ──▶ │  Phase 2:   │ ──▶ │  Phase 3:   │
│  Questions  │     │  Research   │     │  Write      │
└─────────────┘     └─────────────┘     └─────────────┘
                                              │
      ┌─────────────┐     ┌─────────────┐     │
      │  Phase 5:   │ ◀── │  Phase 4:   │ ◀───┘
      │  Publish    │     │  Deaify     │
      └─────────────┘     └─────────────┘
```

| Phase | Что делает | Validation |
|-------|------------|------------|
| 1. Questions | Уточняющие вопросы — угол, контекст, takeaway | answers received |
| 2. Research | Web search, 3-5 источников | sources found |
| 3. Write | Черновик 600+ слов, ASCII диаграмма | word count ≥600 |
| 4. Deaify | 4 параллельных критика | no AI phrases remain |
| 5. Publish | Выбор adapter из config | file saved |

**Все 5 фаз обязательны.**

---

## Phase 1: Questions

**ВСЕГДА начинай с уточняющих вопросов.** Не запускай research пока не получишь ответы.

Спроси у пользователя:

1. **Тема/угол** — о чём конкретно? какой аспект?
2. **Контекст** — личный опыт? решённая проблема? концепция?
3. **Аудитория** — новички/продвинутые? разработчики/вайбкодеры?
4. **Ключевой takeaway** — что читатель должен вынести?

### Если статья про личный опыт

Спроси конкретные цифры:
- Сколько постов/подписчиков/времени?
- Какие результаты?
- Когда это было?

**Лучше статья без цифр, чем статья с враньём.**

---

## Phase 2: Research

**После получения ответов** — проведи research.

Открой новый чат/сессию (или используй subagent) с задачей:

```
Research для блог-поста.

Тема: {тема из Phase 1}
Угол: {ответ пользователя}
Аудитория: {ответ пользователя}

Задачи:
1. Web search по теме (3-5 источников)
2. Найти конкретные примеры, цифры, кейсы
3. Проверить актуальность информации (текущий год)

Верни структурированный research с источниками.
```

**Validation:** минимум 3 источника найдено.

---

## Phase 3: Write Draft

Напиши черновик на основе research:

```
Напиши черновик блог-поста.

Тема: {тема}
Research: {результат Phase 2}

Требования:
- Минимум 600 слов
- Личный тон ("я сделал", "я понял")
- ASCII диаграмма обязательна
- Инлайн-ссылки в тексте (НЕ секция "Источники" в конце)
- Frontmatter: title, date, description, tags

Прочитай `content/writing-guide.md` и `content/ai-terms-ru.md` для стиля.
```

**Validation:** word count ≥ 600, содержит ASCII диаграмму.

---

## Phase 4: Deaify

После получения черновика — **обязательно** прогони через деаификацию.

Прочитай `content/workflows/deaify-text.md` и выполни:
- Запусти 4 параллельных критика
- Проверь факты через web search
- Перепиши с учётом замечаний

**Validation:** все flagged phrases исправлены.

---

## Phase 5: Publish

Прочитай `publish/workflows/publish.md` — он определит adapter из `config/config.yaml`:

- `telegram` — добавит в очередь Telegram канала
- `file-only` — сохранит в папку output/
- `custom` — пользовательский adapter

---

## НИКОГДА не выдумывай факты

**Это критически важно.** Агенты склонны придумывать "реалистичные" детали.

| Нужна деталь | Решение |
|--------------|---------|
| Количество/статистика | Спроси пользователя или оставь `[X постов]` |
| Дата события | Спроси или напиши "недавно", "на днях" |
| Результат/метрики | Спроси или опиши качественно без цифр |

## SEO Requirements

| Поле | Лимит |
|------|-------|
| **title** | ≤60 символов |
| **description** | ≤160 символов |
| **Ссылки** | Инлайн в тексте, не отдельная секция |

Что хотел решить.

Итог.

## ASCII Diagrams

**Обязательна в каждой статье.**

Где использовать:
- **Архитектура** — как компоненты связаны
- **Поток данных** — что куда идёт
- **До/После** — сравнение подходов

Символы: `─ │ ┌ ┐ └ ┘ ├ ┤ ┬ ┴ ┼ ▶ ◀ ▲ ▼`

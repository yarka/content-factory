# Telegram Post Pipeline

Pipeline для написания постов в Telegram-канал.

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  Phase 1:   │ ──▶ │  Phase 2:   │ ──▶ │  Phase 3:   │
│  Questions  │     │  Research   │     │  Write      │
└─────────────┘     └─────────────┘     └─────────────┘
                                              │
          ┌─────────────┐                     │
          │  Phase 4:   │ ◀───────────────────┘
          │  Fact-check │
          └──────┬──────┘
                 │
                 ▼
          ┌─────────────┐
          │  Phase 5:   │
          │  Deaify     │
          └──────┬──────┘
                 │
                 ▼
          ┌─────────────┐
          │  Phase 6:   │
          │  Draft      │
          └─────────────┘
```

**Прочитай `content/channel-dna.md` перед запуском.** Там голос, аудитория, рубрики.

---

## Phase 1: Questions

Получи идею от пользователя. Задай максимум 3 уточняющих вопроса.

**Всегда спрашивай:**
1. **Угол** — что конкретно хочешь сказать? какая мысль?
2. **Контекст** — это из личного опыта? инсайт? реакция на что-то?

**Спрашивай если нужно:**
3. **Рубрика** — к какой из рубрик в `content/channel-dna.md` это относится?

**НЕ задавай больше 3 вопросов.** Лучше додумать и предложить вариант.

---

## Phase 2: Research

**Сразу спроси:**

> "Есть материалы/ссылки по теме или ищем сами?"

### Режим A: Авто-ресёрч

Если пользователь говорит "ищи сам":

```
Найди контекст для поста на тему: {тема}

Задачи:
1. Web search: 2-3 источника (факты, цифры, кейсы)
2. Найди один конкретный пример или историю
3. Проверь актуальность (текущий год)

Верни: 3-5 ключевых фактов/тезисов с источниками.
НЕ пиши черновик — только сырьё.
```

### Режим B: Референсы от пользователя

Если пользователь даёт ссылки или тексты:

```
Прочитай предоставленные материалы.

Извлеки:
1. Ключевые факты и идеи (3-5 пунктов)
2. Цитаты которые можно использовать
3. Угол который ещё не раскрыт в этих материалах

НЕ пиши черновик — только сырьё.
```

### Режим C: Из головы (нет ресёрча)

Если тема — личный опыт или мнение:

> Пропусти фазу research. Спроси у пользователя конкретные детали:
> - Что именно произошло?
> - Какие цифры/даты если есть?
> - Какой вывод?

## Phase 3: Title + Write

### Шаг 3.1 — Сгенерируй 3 варианта заголовка

**ОБЯЗАТЕЛЬНО перед написанием поста.** Заголовок = первая строка = хук.
В Telegram первая жирная строка читается как заголовок.

```
Придумай 3 варианта заголовка для поста.

Тема: {тема из Phase 1}

Используй разные типы из `content/writing-guide.md`:
- Zinger: короткий удар, провокация ("Кодер выживет. Менеджер — нет.")
- First-Person: личный + неожиданный ("Мой запасной план — делать массаж.")
- Question: вопрос-боль ("Если завтра без работы — чем займёшься руками?")
- Scene: картинка момента ("Открываю новости. Notion — больше агентов, чем людей.")

Правила заголовка:
- Максимум 8 слов
- Без точки в конце (кроме намеренного эффекта)
- Не объясняет пост — интригует
- Читается как первая фраза разговора, не как тема письма

Покажи 3 варианта. Жди выбора пользователя.
```

**После выбора заголовка** — переходи к шагу 3.2.

---

### Шаг 3.2 — Напиши черновик поста

```
Напиши пост для Telegram-канала.

Заголовок (выбранный): {заголовок из шага 3.1}
Контекст: {ответы из Phase 1}
Сырьё: {результат Phase 2 или детали из Phase 1}

Прочитай перед написанием:
- content/channel-dna.md → голос, аудитория, рубрики
- content/writing-guide.md → тон, хуки, ритм

ФОРМАТ ПОСТА:
- Первая строка = заголовок (жирный **заголовок**)
- Длина: 200-800 символов (можно до 1500 если тема требует)
- Структура: **Заголовок** → Тело (2-4 абзаца) → Финал (мысль или вопрос)
- Тон: как в `content/channel-dna.md`
- НЕТ хештегов если не указано в `content/channel-dna.md`
- НЕТ ASCII-диаграмм (это не статья)

Верни ТОЛЬКО текст поста. Без комментариев.
```

---

## Phase 4: Fact-check

**Обязательно если пост содержит внешние факты** (цифры, даты, названия продуктов, что компании делают).

Прочитай `content/workflows/fact-check.md` и выполни проверку.

**Инструменты по приоритету:**
1. Exa MCP (`mcp__exa__web_search_exa`) — предпочтительный
2. WebFetch — если дан конкретный URL
3. Web search — fallback

Если пост — личная история или мнение без внешних фактов:
> Напиши "Fact-check: пропущен (нет внешних фактов)" и переходи к Phase 5.

---

## Phase 5: Deaify

**Обязательно.** Прочитай `content/workflows/deaify-text.md` и прогони черновик.

Для Telegram-поста особое внимание на:
- **Critic A:** Generic phrases — "важно понимать", "это позволяет"
- **Critic B:** Ритм — все предложения одинаковой длины = AI
- **Critic C:** Специфика — добавить одну личную деталь

Critic D (факт-чек) — только если в посте есть статистика или даты.

**Hard rule для Telegram:** Пост после deaify должен быть ≤ оригинального по длине.

---

## Phase 6: Draft

Выведи финальный черновик в таком формате:

```
─────────────────────────
ЧЕРНОВИК ПОСТА
─────────────────────────

{текст поста}

─────────────────────────
Символов: {число}
Рубрика: {из content/channel-dna.md}
─────────────────────────

Готово к публикации. Правь или одобряй.
```

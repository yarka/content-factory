## Контекстные правила

1. **Имена продуктов** — всегда на английском: Claude Code, GPT-5, Gemini
2. **Названия техник** — на английском с пояснением при первом упоминании: "Chain-of-Thought (цепочка рассуждений)"
3. **Аббревиатуры** — на английском, расшифровка по желанию
4. **Общие понятия** — предпочитать русский: "рабочий процесс", а не "workflow"
5. **Если сомневаешься** — предпочитай русский вариант

## Примеры трансформации

### Плохо (много англицизмов)
> В моём AI-workflow есть sequential шаг для claim verification. Generic output проходит через rewrite.

### Хорошо
> В моём AI-воркфлоу есть последовательный шаг для проверки утверждений. Шаблонный результат проходит через переписывание.

### Допустимо (баланс)
> В моём воркфлоу есть шаг для проверки утверждений. Промпт проходит через агента.

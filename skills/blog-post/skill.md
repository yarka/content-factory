# Blog Post Pipeline

Модульный pipeline для написания статей: research → draft → deaify → publish.

## Quick Reference

| Что | Где |
|-----|-----|
| **Стиль** | `writing-guide.md` в этой папке |
| **Термины AI** | `ai-terms-ru.md` в этой папке |
| **Деаификация** | skill `deaify-text` |
| **Публикация** | skill `publish` |
| **Конфиг** | `config/config.yaml` |

## Workflow

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  Phase 1:   │ ──▶ │  Phase 2:   │ ──▶ │  Phase 3:   │
│  Questions  │     │  Research   │     │  Write      │
└─────────────┘     └─────────────┘     └─────────────┘
                                              │
      ┌─────────────┐     ┌─────────────┐     │
      │  Phase 5:   │ ◀── │  Phase 4:   │ ◀───┘
      │  Publish    │     │  Deaify     │
      └─────────────┘     └─────────────┘
```

| Phase | Что делает | Validation |
|-------|------------|------------|
| 1. Questions | AskUserQuestion — угол, контекст, takeaway | answers received |
| 2. Research | Task(sonnet) + Exa search | 3+ sources found |
| 3. Write | Task(sonnet) — черновик 600+ слов | word count ≥600, has diagram |
| 4. Deaify | skill `deaify-text` — 4 параллельных критика | no AI phrases remain |
| 5. Publish | skill `publish` — выбор adapter из config | adapter responds |

**Все 5 фаз обязательны.**

## Phase 1: Questions (ОБЯЗАТЕЛЬНО)

**ВСЕГДА начинай с уточняющих вопросов через `AskUserQuestion`.** Не запускай research пока не получишь ответы.

Минимальные вопросы:
1. **Тема/угол** — о чём конкретно? какой аспект?
2. **Контекст** — личный опыт? решённая проблема? концепция?
3. **Аудитория** — новички/продвинутые? разработчики/вайбкодеры?
4. **Ключевой takeaway** — что читатель должен вынести?

Пример `AskUserQuestion`:
```
questions:
  - question: "Какой угол для статьи?"
    header: "Угол"
    options:
      - label: "Личный опыт"
        description: "Я сделал X, получил Y"
      - label: "Tutorial"
        description: "Пошаговая инструкция"
      - label: "Концепция"
        description: "Почему X важно"
```

### Если статья про личный опыт

Спроси конкретные цифры:
- Сколько постов/подписчиков/времени?
- Какие результаты?
- Когда это было?

**Лучше статья без цифр, чем статья с враньём.**

## Phase 2: Research (Subagent)

**После получения ответов** — запусти субагента на Sonnet:

```
Task(model: sonnet, subagent_type: general-purpose):

Research для блог-поста на тему: {тема}

Контекст от пользователя:
- Угол: {ответ}
- Аудитория: {ответ}
- Takeaway: {ответ}

Задачи:
1. Exa search по теме (3-5 источников)
2. Найти конкретные примеры, цифры, кейсы
3. Проверить актуальность информации (текущий год)

Верни структурированный research с источниками.
```

**Validation:** минимум 3 источника найдено.

## Phase 3: Write Draft (Subagent)

Второй субагент пишет черновик:

```
Task(model: sonnet, subagent_type: general-purpose):

Напиши черновик блог-поста.

Тема: {тема}
Research: {результат Phase 2}

Требования:
- Минимум 600 слов
- Личный тон ("я сделал", "я понял")
- ASCII диаграмма обязательна
- Инлайн-ссылки в тексте (НЕ секция "Источники" в конце)
- Frontmatter: title, date, description, tags

Прочитай writing-guide.md и ai-terms-ru.md для стиля.

Верни готовый markdown.
```

**Validation:** word count ≥ 600, содержит ASCII диаграмму.

## Phase 4: Deaify

После получения черновика — **обязательно** прогони через skill `deaify-text`.

Skill запускает 4 параллельных критика:
- Generic phrases detector
- Rhythm analyzer
- Specificity checker
- Fact checker (проверяет версии, даты, ссылки)

Результат: текст без AI-отпечатков, с проверенными фактами.

**Validation:** все flagged phrases исправлены.

## Phase 5: Publish

**Используй skill `publish`** — он прочитает `config/config.yaml` и выберет нужный adapter:

- `telegram` — добавит в очередь Telegram канала
- `file-only` — сохранит в папку output/
- `custom` — твой собственный adapter

Skill publish сам разберётся с деталями.

## НИКОГДА не выдумывай факты

**Это критически важно.** Агенты склонны придумывать "реалистичные" детали:
- "За три месяца опубликовал 9 постов" — откуда эта цифра?
- "Конверсия выросла на 34%" — выдумка
- "Сделал за полчаса" — не знаешь точно

### Что делать вместо выдумок

| Нужна деталь | Решение |
|--------------|---------|
| Количество/статистика | Спроси через AskUserQuestion или оставь `[X постов]` |
| Дата события | Спроси или напиши "недавно", "на днях" |
| Результат/метрики | Спроси или опиши качественно без цифр |
| Личный опыт | Опиши проблему абстрактно, без конкретных чисел |

## SEO Requirements

| Поле | Лимит |
|------|-------|
| **title** | ≤60 символов |
| **description** | ≤160 символов |
| **Ссылки** | Инлайн в тексте, не отдельная секция |

**Перед сохранением** проверь длину:
```python
title = "..."
desc = "..."
print(f"Title: {len(title)}/60, Desc: {len(desc)}/160")
```

## ASCII Diagrams

**Обязательна в каждой статье.** Диаграммы делают текст нагляднее.

Где использовать:
- **Архитектура** — как компоненты связаны
- **Поток данных** — что куда идёт
- **До/После** — сравнение подходов
- **State machine** — состояния и переходы

Символы: `─ │ ┌ ┐ └ ┘ ├ ┤ ┬ ┴ ┼ ▶ ◀ ▲ ▼`

## Common Mistakes

| Ошибка | Исправление |
|--------|-------------|
| **Статья слишком короткая** | Минимум 600 слов. Добавь контекст, альтернативы |
| **Нет ASCII диаграммы** | Добавь визуализацию |
| Пропустил deaify | Текст звучит как AI. Всегда прогоняй |
| Нет ссылок | Exa research обязателен |
| Секция "Источники" внизу | Ссылки инлайн |
| Длинный хук | Максимум 30 слов |
